# AGENTS.md

## Purpose
This repository contains a landing page used to share the 2026 5th Grade Slide Show with families, students, teachers, and school staff.

## Primary objectives
- Provide a clear, welcoming landing page.
- Make the slideshow link easy to find and use.
- Keep the page fast, accessible, and mobile-friendly.
- Minimize maintenance burden by keeping the codebase simple.

## Expected user experience
- Visitors should understand the page purpose immediately.
- The main call to action should be obvious.
- Important information should be readable on phones, tablets, and desktops.
- Navigation and interactions should be simple and predictable.

## Guardrails
- Do not include private student information.
- Do not add unnecessary frameworks or dependencies without a clear benefit.
- Do not over-engineer features for a simple landing page.
- Keep copy school-appropriate, concise, and friendly.

## Preferred implementation style
- Use semantic HTML where possible.
- Keep CSS organized and easy to adjust.
- Use JavaScript only when it adds clear value.
- Prefer small, focused edits over large refactors.
- Preserve the current structure unless a change is requested.

## Content expectations
- Use a warm and welcoming tone.
- Favor short headings and short paragraphs.
- Make calls to action specific and easy to scan.
- Avoid filler text in production content.

## If requirements are unclear
When making updates:
1. Choose the simplest reasonable implementation.
2. Leave brief TODO notes only when necessary.
3. Document assumptions in summaries, commits, or pull requests.
